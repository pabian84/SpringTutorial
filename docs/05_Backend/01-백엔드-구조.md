# 백엔드 구조 및 아키텍처

## 1. 백엔드 기술 스택
- **Framework**: Spring Boot 3.x
- **Language**: Java 17+
- **Security**: Spring Security 6, JJWT
- **Database**: H2 (Embedded), MyBatis
- **Real-Time**: Spring WebSocket

## 2. 주요 패키지 설계 (`server/src/main/java/com/example/demo/`)

### `domain/` (비즈니스 로직)
애플리케이션의 핵심 기능을 도메인별로 분리한 패키지 구조(DDD 지향)입니다.
- **`auth/`**: 로그인 요청 처리 및 JWT 토큰 발급/검증.
- **`user/`**: `UserController`, `UserService`, `SessionService`. 사용자 정보 조회, 세션 조회(`user_sessions` DB 연동) 및 강제 로그아웃 기능.
- **`chat/`, `memo/`, `weather/`**: 개별 위젯의 데이터를 제공하고 저장하는 REST API 담당.

### `global/` (전역 설정)
- **`config/`**: 
  - `SecurityConfig.java`: 엔드포인트 권한 설정, CORS, JWT 필터 체인 추가.
  - `WebSocketConfig.java`: STOMP 엔드포인트 및 브로커 설정.
- **`security/`**: `JwtTokenProvider.java` 및 커스텀 인증 필터(`JwtAuthenticationFilter.java`). 모든 HTTP 요청의 쿠키를 가로채어 유효성을 검사합니다.
- **`exception/`**: 애플리케이션 전체에서 발생하는 예외를 가로채어 일관된 JSON 형식(CustomException)으로 응답합니다.

### `handler/` (웹소켓)
- **`WebSocketHandler.java`**: 커스텀 웹소켓 핸들러. 연결된 클라이언트의 세션을 관리하며, 새 기기 로그인 알림이나 강제 연결 종료(`FORCE_LOGOUT`) 메시지를 송신합니다.

## 3. 핵심 보안 로직: JWT & Device ID
* **`JwtTokenProvider`**: 로그인 시 2개의 쿠키(`accessToken`, `deviceId`)를 Response 헤더에 세팅합니다.
### `SessionService` (세션 및 웹소켓 관리)
`user_sessions` 테이블의 비즈니스 로직을 담당하며, 실시간 통신 보안의 핵심 브리지 역할을 합니다.
* **데이터 일관성**: HTTP 요청을 통해 세션이 무효화될 때(`deleteSession`, `deleteAllSessions` 등), DB뿐만 아니라 현재 연결된 **실시간 웹소켓 세션까지 동기적으로 종료**합니다.
* **보안 Close Code (4001)**: 웹소켓 종료 시 `4001` 코드를 사용하여 클라이언트에게 단순 연결 오류가 아닌 '강제 로그아웃'임을 명시적으로 전달합니다.
* **이벤트 기반 알림**: `UserService.login` 성공 시 `NewDeviceLoginEvent`를 수신하여 다른 기기에 새 로그인 알림을 실시간 브로드캐스트합니다.
