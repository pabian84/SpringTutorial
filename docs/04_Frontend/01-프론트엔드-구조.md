# 프론트엔드 구조 및 아키텍처

## 1. 프론트엔드 기술 스택
- **Framework**: React 18
- **Build Tool**: Vite 5
- **Language**: TypeScript 5
- **Routing**: React Router DOM 6
- **Styling**: Tailwind CSS
- **Visualization**: React Grid Layout, Three.js, CesiumJS

## 2. 주요 디렉토리 설계 (`client/src`)

### `api/`
Axios 인스턴스를 통해 백엔드(Spring Boot)의 RESTful 엔드포인트와 통신하는 함수 모음입니다.
- `sessionApi.ts`: 기기 세션 강제 종료 등
- `userApi.ts`: 로그인/로그아웃, 사용자 정보 확인

### `components/`
도메인 또는 재사용성 단위로 분리된 React 컴포넌트입니다.
- **`cesium/`, `threejs/`**: 3D 렌더링에 특화된 시각화 컴포넌트 래퍼.
- **`Servermonitor.tsx`, `ChatWidget.tsx`, `MemoWidget.tsx`**: 대시보드 내 개별 위젯 컴포넌트.

### `contexts/` (Global State)
- **`AuthContext.tsx`, `AuthProvider.tsx`**: 가장 핵심이 되는 인증 상태 관리자입니다. 로그인 여부, 사용자 정보, `checkAuthStatus`를 전역으로 공급합니다.
- **`WebSocketContext.tsx`**: 실시간 채팅 및 서버 모니터링 데이터를 받아오고 재연결 로직을 관리합니다.

### `utils/`
- **`authUtility.ts`**: 인증 관련 이벤트(`AUTH_EVENTS`), 로컬스토리지 제어, 다중 탭 동기화 로직 등을 추상화.
- **`axiosConfig.ts`**: Axios의 요청/응답 인터셉터를 정의. 401(토큰 만료) 에러 시 `refreshToken` 로직을 자동으로 수행하고 실패 시 `REQUEST_LOGOUT` 이벤트를 발행합니다.

## 3. 핵심 로직: 로그인 상태 관리 (레이스 컨디션 방어)
React의 상태 업데이트 지연(Render gap)으로 인해 발생하는 "불필요한 API 중복 호출 및 토스트 발생" 이슈를 막기 위해 다음 전략이 적용되어 있습니다.
1. `isAuthenticatedRef`: 즉각적인 상태 확인용.
2. `authState`: UI 렌더링용 상태.
3. 컴포넌트 내부에서 `checkAuthStatus()`의 무분별한 호출을 제한하고, 라우터 진입 및 앱 마운트 시점에만 확인합니다.
