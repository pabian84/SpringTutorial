## 핵심 개념

### 인증 흐름

```
로그인 -> JWT 토큰 (HttpOnly 쿠키) -> API 요청 -> 로그아웃
```

참고: [02-시스템-아키텍처.md](02-시스템-아키텍처.md#2-인증-아키텍처)

### 기기 관리

```
세션 조회 -> 기기 강퇴 -> 강제 로그아웃 (웹소켓 4001)
```

참고: [05-시나리오-다이어그램.md](05-시나리오-다이어그램.md#3-기기-관리-시나리오)

### 토큰 만료

```
API 호출 -> 401 응답 -> 인증 확인 -> 유효하지 않으면 로그아웃
```

참고: [05-시나리오-다이어그램.md](05-시나리오-다이어그램.md#6-토큰-만료-시나리오)

## 아키텍처 요약

```
프론트엔드 (React + TypeScript)
    |
    |-- HTTP/REST --> 백엔드 (Spring Boot)
    |-- 웹소켓 --> 웹소켓 핸들러
    |
백엔드
    |
    |-- MyBatis --> H2 데이터베이스
    |-- JWT --> HttpOnly 쿠키
```

참고: [02-시스템-아키텍처.md](02-시스템-아키텍처.md)

## API 요약

| 도메인 | 엔드포인트 |
|--------|-----------|
| 인증 | GET /api/auth/check |
| 사용자 | POST /api/user/login, POST /api/user/logout, GET /api/user/logs |
| 세션 | GET /api/sessions, POST /api/sessions/revoke, DELETE /api/sessions/others, DELETE /api/sessions/all |
| 날씨 | GET /api/weather |
| 메모 | GET /api/memo/{userId}, POST /api/memo, DELETE /api/memo/{id} |
| 채팅 | GET /api/chat/history |
| 통계 | GET /api/stats/code |

참고: [04-API-명세서.md](04-API-명세서.md)
